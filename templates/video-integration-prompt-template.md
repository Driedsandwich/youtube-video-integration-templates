# YouTube動画統合プロンプトテンプレート

> 技術系YouTube動画の内容をプロジェクトに統合するための標準プロンプト

## 📋 使用方法

1. このテンプレートをコピー
2. `[変数]` を実際の値に置き換え
3. Cursor Agentに貼り付けて実行

---

## 📋 実装手順（AI Agent向け）

### Phase 1: 分析（5-10分）
1. **動画トランスクリプト取得**
   - `mcp_fetch_fetch_youtube_transcript` を使用
2. **内容分析**
   - 主要テーマの抽出
   - 実践的知見の整理
   - プロジェクトとの関連性評価
3. **プロジェクト構造把握**
   - 既存ファイルの確認
   - 命名規則・スタイルの理解

### Phase 2: 計画（2-5分）
1. **TODOリスト作成**
   - `todo_write` ツールで管理
   - 各タスクに優先度設定
2. **ファイル構成の設計**
   - 配置場所の決定
   - ファイル名の決定
   - 相互リンクの計画

### Phase 3: 実装（20-40分）
1. **ドキュメント作成**
   - 初心者向けガイド
   - 技術詳細ドキュメント
2. **ルール追加**
   - `.cursor/rules/` 配下
3. **設定ファイル更新**
   - `.gitignore`, `.env.example`等
4. **既存ドキュメント拡張**
   - 統合運用ガイド等に追加
5. **テンプレート改善**
   - PR/Issueテンプレート

### Phase 4: 統合（5-10分）
1. **クロスリンク追加**
   - README更新
   - 関連ドキュメント間のリンク
2. **整合性確認**
   - リンク切れチェック
   - 命名規則の確認

### Phase 5: 完了報告（2-5分）
1. **成果物のサマリー**
2. **次のステップの提案**

## 💡 使用例

### 例1: Git/GitHub入門動画の統合
```markdown

## プロジェクトコンテキスト
- プロジェクト名: PIMS
- 主要技術: GitHub Actions, Notion API
- 既存ドキュメント: docs/, development-docs/

## 統合の目的
AI駆動開発における安全なGit運用を確立したい。
非エンジニアメンバーもGitを使えるようにしたい。
```
